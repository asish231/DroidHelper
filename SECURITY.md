# Security Policy

## Supported Versions

Security fixes are handled on the latest code in `main`.

## Reporting a Vulnerability

Please do not open public issues for suspected security vulnerabilities.

Instead:

- contact the repository owner privately through GitHub
- include clear reproduction details
- include impact and affected environment if known

I will review valid reports and respond as quickly as practical.

## Scope Notes

This project wraps external tools such as `adb` and `scrcpy`.

When reporting an issue, please distinguish between:

- a problem in DroidHelper itself
- a problem originating from `adb`
- a problem originating from `scrcpy`
- an operating-system trust or code-signing prompt
